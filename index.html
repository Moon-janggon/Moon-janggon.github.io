프로젝트 1: 스마트 오피스 AIoT 환경 모니터링

기간: 2024.09 ~ 2025.06 NHN Academy AIoT 2기 부트캠프 | 팀 프로젝트

요약: Spring 및 Iot 센서로 연결된 서버를 모니터링하는 APM 서비스입니다.

기술 스택: Java, Spring Boot, MSA, MySQL/InfluxDB, Docker

  • 주요 기능 및 당신의 역할: IoT 센서 데이터 수집 및 처리 모듈 개발: 다양한 IoT 센서(온도, 습도, 전력 등)로부터 실시간 데이터를 수집하기 위한 Spring Boot 기반의 RESTful API 엔드포인트를 설계 및 구현했습니다. 초당 수백 건 이상의 센서 데이터를 안정적으로 처리하기 위해 Spring WebFlux의 비동기 처리 방식과 Reactor 라이브러리를 활용하여 고성능 데이터 수신 모듈을 개발했습니다. 수집된 데이터의 유효성을 검증하고, 필요한 경우 데이터 정규화 및 초기 가공 로직을 적용했습니다. 데이터 파이프라인 구축 및 저장소 관리: 수집된 IoT 센서의 시계열 데이터를 효율적으로 저장하고 조회하기 위해 InfluxDB를 메인 시계열 데이터베이스로 선정하고 연동했습니다. 이를 통해 대용량 시계열 데이터의 고속 쓰기 및 범위 쿼리를 최적화했습니다. 센서 메타데이터, 사용자 정보, 모니터링 설정 등 관계형 데이터는 MySQL에 저장하도록 데이터베이스 스키마를 설계하고 관리했습니다. 데이터 결측치 발생 시, 이전 데이터와의 선형 보간(Linear Interpolation) 또는 이동 평균(Moving Average) 방식을 적용하여 데이터 신뢰성을 확보하는 파이프라인을 구축했습니다. API 설계 및 배포 담당: 마이크로서비스 아키텍처 원칙에 따라 각 서비스(데이터 수집, 분석, 알림 등) 간의 명확한 책임 분리 및 RESTful API 명세를 정의했습니다. 서비스 간 통신은 Spring Cloud OpenFeign을 활용하여 선언적 HTTP 클라이언트를 구현, 코드의 가독성과 유지보수성을 높였습니다. 개발된 각 마이크로서비스는 Docker 컨테이너 이미지로 빌드하고 최적화된 Dockerfile을 작성하여 환경 독립적인 배포가 가능하도록 했습니다. GitHub Actions를 활용하여 CI/CD(지속적 통합/지속적 배포) 워크플로우를 구축했습니다. 코드 푸시 시 자동 빌드, 테스트, Docker 이미지 생성 및 원격 서버(AWS EC2)로의 배포를 자동화하여 개발 생산성과 배포 안정성을 확보했습니다.
  • 문제 해결 및 배운 점: 대규모 IoT 센서 데이터 및 서버 리소스의 안정적인 수집 및 결측치 처리: 초당 발생하는 대량의 센서 데이터를 처리하며 발생하는 성능 병목 현상을 해결하기 위해 비동기 처리와 시계열 데이터베이스(InfluxDB)의 특성을 깊이 이해하고 적용했습니다. 특히, 불완전한 센서 데이터의 결측치를 효과적으로 보정하는 로직을 구현하며 데이터 신뢰성 확보의 중요성을 체감했습니다. 마이크로서비스 아키텍처 설계 및 원격 서버 배포 역량 확보: 서비스 간 결합도를 낮추고 독립적인 확장이 가능한 MSA를 설계하는 과정에서, 각 서비스의 역할과 경계를 명확히 정의하는 것이 중요함을 배웠습니다. 또한, Docker를 활용한 컨테이너화와 GitHub Actions를 통한 CI/CD 파이프라인 구축 경험은 실제 서비스 환경에 애플리케이션을 배포하고 관리하는 실무 역량을 크게 향상시켰습니다. 이 과정에서 발생한 네트워크 설정, 환경 변수 관리 등의 다양한 배포 이슈를 해결하며 문제 해결 능력을 길렀습니다.
